如何使用Hexo搭建Github Pages博客

前端 github pages hexo 2019/09/19

1. 安装Node.js及Git

具体方法参见 Node.js 及 Git 官网教程。
如果本地已安装好可跳过此步。

2.新建一个文件夹，用于存放博客代码

我这里给这个新建的文件夹命名为：githubhomepage

3.安装Hexo

在命令行中运行以下命令安装Hexo

4.使用命令行，创建Hexo博客项目

进入刚才创建的博客文件夹githubhomepage中 ，执行初始化博客命令，并安装博客所需的包

5.启动server

6.启动成功后在浏览器中运行

默认本地运行地址是http://localhost:4000

7.创建博客用的git仓库

仓库名称必须为 username.github.io

注：username为你在github的用户名

假设github用户名为john，那你的仓库就必须是john.github.io。如果你创建一个叫做jason.github.io的仓库，最后deploy完以后打开页面是会报错的。

8.在Hexo中配置项目部署信息

在博客文件夹根目录下，找到文件_config.yml。在deploy部分填写配置信息，把repository值设置为你第7步中创建的博客用的git仓库地址。

9.部署到Github Pages

如果上面这个步骤报错：ERROR Deployer not found : git，尝试安装hexo-deployer-git

完成后再次执行hexo deploy命令，应该就可以解决问题了。

部署成功后，打开http://your-github-username.github.io就能看见默认博客。

比如我的博客的地址是:http://johnnynie.github.io

至此，hexo的博客生成和部署已经全部成功。

10.新建blog文章

在自己的blog目录下，运行下面的命令便可以快速创建一篇新的blog文章

注：
post是默认的layout，title是新文章的md文件名和url对应地址。

title不要带空格，如果有空格，创建文件的时候文件名只会是最后一个空格后面的文字。它将会用在该post的url中，而且如果_config.yml中的post_asset_folder true（是否生成与每篇文章对应的资源文件夹，默认值为false），资源文件夹也会直接使用title值，所以建议最好使用英文来创建。真正的文章题目可以在md文件中修改。

文章创建成功后，在 /source/_posts 文件夹下，会生成一个新的markdown文件，文件名就是你刚刚的title。

11.将新文章或者更改发布到你的github主页

注意：因为前面第9步中已经发布过一次了，所以后面需要再次发布的话，需要在发布前运行hexo clean清空构建文件夹及运行hexo generate命令再次构建所需的文件

如下：

执行完后，再次运行hexo deploy,就可以再次发布成功了。

可以看到，如果按照此步骤的话，每次在运行hexo deploy命令前，都需要运行hexo clean及 hexo generate命令，这样显然有点麻烦，那么有没有更好的方式呢？答案是有的。

我们可以在命令行中运行 hexo generate –watch命令，这样有新增或修改博客文章时就会实时构建了

这样，我们就不需要再在运行hexo deploy命令前，运行hexo clean及 hexo generate命令了。

以上就是使用Hexo搭建Github Pages博客及发布文章的全过程了。
